# 2.2. 不具合によるリスク

# 2.2.1. 信頼性に関する課題

## 主要情報

- 汎用AIシステムは、失敗すると害をもたらしうる。失敗には、虚偽または捏造された情報の生成（しばしば「ハルシネーション」と呼ばれる）、欠陥のあるコンピューターコードの記述、誤解を招く医療上の助言の提供が含まれる。これらの失敗は、身体的または心理的な害をもたらし、利用者や組織を評判上の損害、金銭的損失、法的責任にさらす可能性がある。
- モデルの挙動はしばしば理解または予測が困難であり、信頼性を保証することを困難にしている。汎用AIモデルの開発者でさえ、しばしばモデルの挙動を意味のある形で説明したり、特定の失敗モードを予期したり、そのような失敗が起こらないことを実証したりできない。悪意のある行為者はまた、AIの開発に干渉したり、安全対策を回避する敵対的な入力をシステムに与えたりすることによって、失敗を引き起こすこともできる。
- AIエージェントは、自律的に行動し、他のシステムや物理世界に直接影響を与えうるため、高まった信頼性リスクをもたらす。人間が介入する機会が少ないため、エージェントの失敗はより大きな害をもたらしうる。マルチエージェントシステムは、エージェント間の相互作用を通じて誤りが伝播・増幅されうるため、さらなるリスクをもたらす。
- 前回の報告書の公表（2025年1月）以降、AIシステムは概してより信頼性が高くなり、その結果、商業展開が大幅に増加した。ハルシネーションのような多くの種類の失敗は概して起こりにくくなっているが、システムはより複雑なタスクを遂行する際には依然として一般的に誤りを犯す。
- 著しい研究努力にもかかわらず、いかなる手法の組み合わせも、重要な分野で必要とされる高い信頼性を保証しない。新しい訓練方法とAIシステムへのツールへのアクセスの付与は、失敗を起こりにくくすることができるが、通常はそれを完全には排除しない。

汎用AIシステムは、捏造された法的引用から医療上の誤診まで、既に実世界での害を引き起こしている方法で失敗する。人間の専門家も誤りを犯すが、AIの失敗は、その新規性、潜在的な規模、いつ発生するかを予測することの困難さ、そして自信ありげに聞こえる出力を無批判に信頼する利用者の傾向のために、独特の懸念を提起する。現在の汎用AIの失敗には、虚偽の情報を提供すること（602, 603）、基本的な推論の誤りを犯すこと（604, 605）、新しい文脈で展開された際に性能が劣化すること（606, 607, 608）が含まれる。そのような失敗による文書化された害には、医療の誤診、法的準備書面の誤り、金銭的損失が含まれる（609, 610, 611）。信頼性の課題は、AIエージェントにとって特に重要である。なぜなら、失敗は人間の行為や監督なしに直接害を引き起こしうるからである（612*, 613, 614, 615*）。マルチエージェントシステムは、協調不全、対立、あるいはエージェント間の望ましくない結託を通じて、さらなる失敗モードをもたらす（614, 616）。

## 汎用AIシステムは様々な信頼性の課題に直面している

Table 2.4は、信頼性の問題の一般的な区分を要約している。最初の3つはすべてのAIシステムに当てはまる一方、最後の2つは特にAIエージェントとマルチエージェントシステムに関連する。多くの信頼性リスクは、AIシステムの挙動を予測しモニタリングすることの困難さに起因する。これらの課題（§3.1. 技術的・制度的課題 でさらに論じる）は、複雑な環境で動作するAIエージェントにとって特に深刻である。そのような失敗を評価し緩和するための現在の技術は失敗率を減らすことができるが、最先端のAIエージェントでさえ、依然として十分に信頼性が低く、リスクをもたらし、多くの文脈での展開を妨げている。

「信頼性」とは、AIシステムが開発者または利用者によって意図された通りに機能する度合いを指す。汎用AIシステムは、不正確または誤解を招くコンテンツの生成から、基本的な推論の遂行における失敗まで、様々な信頼性の問題を経験する。例えば、モデルは事実情報の想起において改善しているが、最先端のモデルでさえ、依然として著しい割合で自信ありげだが不正確な回答をする（Figure 2.10）。ソフトウェアエンジニアリングにおいて、汎用AIは現在、コンピューターコードの記述、評価、デバッグにおいて相当な支援を提供できる（215*, 628, 629）。しかし、AI生成コードにはしばしばバグが含まれ（630）、コーディングエージェントは定期的に誤りを犯す（631）。そのような失敗は、プログラムやセキュリティシステムに脆弱性をもたらしうる（§2.1.3. サイバー攻撃 参照）。

**Table 2.4: 汎用AIシステム、AIエージェント、マルチエージェントシステムにおける文書化された信頼性の問題**

| 信頼性の問題 | 例 |
|---|---|
| ハルシネーション | 法的準備書面において存在しない判例を引用すること（617）／遺族の乗客のために存在しない割引運賃方針を引用すること（618）／不正確でバイアスのかかった医療情報を提供すること（619）／出来事についての時代遅れの情報を提供すること（620） |
| 基本的な推論の失敗 | 数学的計算を実行できないこと（621）／基本的な因果関係を推論できないこと（622*） |
| 分布外失敗（不慣れな、または異常な入力への失敗） | 背景の照明や文脈が変化した際に画像を誤分類すること（623） |
| ツール利用の失敗 | 利用者の非公開画像を第三者のツールに送信するAIエージェントによるプライバシー侵害（624）／短期作業記憶の失敗（625, 626） |
| マルチエージェントシステムの失敗：協調不全と対立 | 個々のインセンティブと集団の福祉目標との間の対立のために、共有資源を管理できないこと（627） |

*Table 2.4: 汎用AIシステム、AIエージェント、マルチエージェントシステムにおける文書化された信頼性の問題。*

信頼性の問題は、AIの利用の加速と、失敗が深刻な害をもたらす可能性のために、医療のようなリスクの高い環境において特に追跡することが重要である（609, 619）。関連する能力は急速に向上しており、最先端のモデルは現在、医療試験に合格できる（633*, 634）。しかし、実世界での利用は、ベンチマークが見逃す限界を明らかにする。例えば、ある研究では、モデルは提起された医療質問の19%に対して潜在的に有害な回答を提供した（635）。そのような失敗は、誤診、不適切な治療、あるいは不当な医療拒否をもたらしうる（611）。

## AIエージェントはその自律性ゆえに新規の信頼性リスクをもたらす

AIエージェントは実世界において直接行動するため、その失敗は非エージェント型システムにおける失敗よりも大きな害をもたらす可能性がある（99）。人間がレビューするためにテキストや画像を単に生成するAIシステムとは異なり、AIエージェントは、世界に影響を与える行為を独立して取ることができる（99, 615*, 636, 637）（§1.1. 汎用AIとは何か も参照）。AIエージェントは、行為を開始し、他の人間やAIシステムに影響を与え、将来の結果を動的に形作ることができる。この影響の範囲の拡大は、新たなリスクをもたらし、信頼性の重要性を増幅させる。なぜなら、失敗は、人間が介入する機会なしに直接害を引き起こす可能性があるからである（99, 612*, 638, 639, 640）。これは、金融サービス（641）、エネルギー管理（642）、あるいは科学研究（643*, 644）といった戦略的または安全上重要な環境に展開されるエージェントにとって特に重要である可能性がある。

## マルチエージェントAIシステムは新しい種類の信頼性の失敗をもたらす

マルチエージェントAIシステムは、協調の失敗やエージェント間の対立により、新しい種類の信頼性の失敗をもたらす。マルチエージェントAIシステムにおいて、エージェントは、共有された目標または個々の目標を追求しながら互いに相互作用する（614, 645, 646, 647, 648, 649）。例えば、研究文献レビューを行うよう設計されたマルチエージェントシステムにおいて、主導エージェントは利用者の問い合わせを分解し、それぞれが異なる側面を並行して調査する専門の下位エージェントに下位タスクを割り当てる（650*）。これは効率性の向上を可能にする一方、誤りがエージェント間で伝播しうることも意味する（614, 651, 652, 653, 654, 655）。複数のエージェントが同じベースモデルに基づいているか、同じツールを組み込んでいる場合、それらはまた相関した失敗を示す可能性がある（656）。展開されたシステムにおけるそのような失敗の実証的証拠は依然として限られているが、マルチエージェントシステムがより一般的になるにつれて、これらのリスクは増大する可能性がある。

## 更新情報

前回の報告書の公表（2025年1月）以降、AIエージェントへの商業的・研究的関心は大幅に増加した。より多くのAIエージェントが実世界で展開されており（Figure 2.11）、その大部分はコンピューター利用またはソフトウェアエンジニアリングの応用に特化している（92）。XBOWハッキングエージェント（467*）、Claude-4（659*）、ChatGPT Agent（660*）のような最近のリリースは、Web検索に基づいてスライドデッキを作成するといった、萌芽的な自律的能力を実証している（660*）。しかし、それらは、タスクが長くなるにつれて失敗率が上昇するため（98, 148）、旅行の計画と予約のような、より複雑なタスクをまだ遂行できない（100*）。現在の研究には、エージェントが外部ツールや他のエージェントとどのように通信するかについての標準を開発する取り組みが含まれる（661, 662）。例には、GoogleのAgent2Agent（663*）とAgent Payments（664*）プロトコル、およびAnthropicのModel Context Protocol（665*）が含まれる。

本節は意図しない信頼性の失敗に焦点を当てているが、悪意のある行為者は、プロンプトインジェクションのような攻撃を通じて意図的に失敗を引き起こすこともできる。プロンプトインジェクション攻撃では、悪意のある指示が、ウェブサイトやデータベース内の隠された指示のような経路を通じて間接的にエージェントに提示される（507, 657, 658）。これらの指示は、エージェントを「乗っ取り」、利用者の意図に反して行動させる可能性がある。そのような攻撃は、利用者や開発者の制御外にある外部のコンテンツを使用して行われるため、防御することが特に困難である。攻撃の標的としてのAIシステムについては §2.1.3. サイバー攻撃 でさらに論じ、技術的な防御については §3.3. 技術的安全対策とモニタリング で扱う。

## 証拠のギャップ

主要な証拠のギャップは、AIシステムの能力、限界、失敗モードを確実に評価することの困難さに起因する（§3.1. 技術的・制度的課題 参照）。AIエージェントの信頼性の体系的な評価は限られており、標準化を欠いている（92, 666）。時代遅れの情報への依存（620）のような特定の問題は、実世界での利用においてのみ現れる可能性があり、展開前評価を不十分なものにしている。以前の研究は、従来のソフトウェアと初期の形態のAIにおけるエージェントとマルチエージェントシステムの信頼性を検討してきた（647, 667, 668）。しかし、この研究が、しばしば大規模言語モデルに基づく現代のAIエージェントにどれほど適用可能であるかは不明である（669）。一部の研究者は、エージェントが互いの相互作用において示しうる、結託や相関した失敗といった新規の挙動について懸念を提起しているが（614）、実証的証拠は依然として限られている。これらのギャップに対処する取り組みには、米国国立標準技術研究所（NIST）による新しいエージェント乗っ取りリスクの評価（670）、OECDのAI Capability Indicators（243）、英国AI Security InstituteのInspect Sandboxing Toolkit（671）が含まれる。

## 緩和策

AIの信頼性を改善する技術は、モデル自体と、それが展開されるより広いシステムの両方を対象としている。これらは失敗率を減らすことができるが、重要な分野で必要とされる高い信頼性をまだ保証できるものはない（672）。重要な技術的手段は、訓練中にモデルを困難な入力にさらすことで、より適切で頑健な応答を発達させるのを助ける敵対的学習である（673, 674, 675, 676, 677）（§3.3. 技術的安全対策とモニタリング 参照）。ハルシネーションを減らすため、開発者は、モデルの応答を外部データベースから取得した情報で補完し、出力が正確かつ最新であることを確実にするのに役立つ検索拡張生成（RAG）を適用したり（678, 679, 680）、より事実に即した（681）、あるいはより効果的に推論する（682）ようモデルを特に微調整したりすることができる。環境ベースまたはツールベースの方法もまた、開発者がAIシステムをモニタリングするのに役立ちうる（683）。例えば、デプロイヤーは、より広く展開する前に潜在的な失敗モードを分析するために、限定されたサンドボックス化された環境でAIシステムを試験的に運用することができる。

AIエージェントについては特に、研究者は、改善された透明性、監督、モニタリングを通じて信頼性を改善することを提案している。例えば、エージェントの外部ツールや他のエージェントとの相互作用をモニタリングすることは、エージェントの活動のより効果的な監督（684, 685）とインシデント分析（686）を可能にするだろう。マルチエージェントの設定を含め、そのような情報を自動的に収集する方法は、依然として活発な研究分野である（653, 654）。

## 政策立案者にとっての課題

政策立案者にとっての主要な課題には、AIエージェントの展開の利益を信頼性の失敗のリスクと比較検討すること、そして開発者、デプロイヤー、利用者がエージェントの性能とリスクプロファイルについての正確な情報にアクセスできることを確実にすることが含まれる。AIエージェントによって引き起こされた害についての責任をどのように帰属させるかを決定することは、さらなる課題をもたらす（639）。特に、失敗がいつどのように起こったかを特定することが難しい可能性があるマルチエージェントの設定においてはそうである（687）。これらの課題は、エージェントが自律性と外部ツールへのアクセスを獲得するにつれて、エージェントの信頼性を評価することの困難さによって複合化される（688*, 689）。エージェント的能力がどれほど急速に出現するかについての不確実性もまた、新規の課題への計画を困難にする（「証拠のジレンマ」については §3.1. 技術的・制度的課題 を参照）。
