# 用語集 (Glossary)

International AI Safety Report 2026 の翻訳で使用する統一用語集。原典 Glossary 章（p.147-155）の全180項目を収録している。

## 使い方

- 本文中で専門用語が**初出**する箇所では「日本語訳（English term）」の形で英語を併記する。
- 2回目以降の出現では日本語訳のみを用い、訳語を統一する。
- 新しい専門用語が本文中に見つかった場合は、この表に追加してから翻訳を進める。
- 定義文（日本語）は原典 Glossary の定義を正確に翻訳したものであり、本文中の用語の意味を確認する際の参照として用いる。

| English Term | 日本語訳 | 定義（日本語） |
|---|---|---|
| Adoption | 普及 | 個人や組織が、業務や日常の実践の中で新しい技術を使い始めること。 |
| Adversarial training | 敵対的学習 | モデルの信頼性を高めるために用いられる機械学習の手法。まず開発者がモデルを失敗させるように設計した入力を作成し、次にそうした入力を認識し適切に扱えるようモデルを訓練する。 |
| AI agent | AIエージェント | ファイルの作成やウェブ上での行動、他のエージェントへのタスクの委任など、環境と適応的に相互作用しながら複雑なタスクを遂行し、人間の監督をほとんど、あるいは全く受けずに目標を追求できるAIシステム。 |
| AI companion | AIコンパニオン | 感情的な支えの提供など、利用者と個人的な関係性をシミュレートするよう設計されたAIシステム。 |
| AI developer | AI開発者 | AIモデルやシステムを設計、構築、または改変する組織。 |
| AI-enabled biological and chemical tools | AI活用型生物・化学ツール | 生物学的または化学的データで訓練され、科学的応用における有用性を高めた専門特化型AIモデル。 |
| AI exposure | AIエクスポージャー（AIへの曝露度） | 特定の業務活動や職種が、人間の能力の拡張あるいはタスクの自動化を通じて、AIシステムによってどの程度影響を受けうるかの度合い。 |
| AI-generated media | AI生成メディア | 生成AIによって作成された音声、テキスト、または映像コンテンツ。 |
| AI lifecycle | AIライフサイクル | データ収集・整備、事前学習、事後学習・ファインチューニング、システム統合、展開・リリース、展開後のモニタリングと更新など、AI開発の各段階。 |
| Algorithm | アルゴリズム | AIシステムがデータを処理し特定のタスクを遂行することを可能にする一連の規則または命令。 |
| Algorithmic efficiency | アルゴリズム効率 | 使用メモリ量や学習にかかる時間など、アルゴリズムがデータから学習する際に用いる計算資源の量を示す一連の指標。 |
| Algorithmic transparency | アルゴリズムの透明性 | 推奨や意思決定といった汎用AIの出力に影響を与える要因が、各ステークホルダーにとってどの程度知りうるものであるかの度合い。こうした要因には、AIモデルの内部の仕組み、その訓練方法、訓練に用いられたデータ、入力のどの特徴が出力に影響したか、異なる状況下ではどのような判断を下していたかなどが含まれうる。 |
| Alignment | アライメント | AIモデルやシステムが、その能力を人間の意図、価値観、または規範に沿って用いる性質。文脈によっては、開発者、利用者、特定のコミュニティ、あるいは社会全体など、さまざまな主体の意図や価値観を指す場合がある。 |
| Application programming interface (API) | アプリケーション・プログラミング・インターフェース（API） | ソフトウェアアプリケーション間の統合と通信を可能にする一連の規則とプロトコル。例えば、AIシステムと検索エンジンの間の連携など。 |
| Artificial general intelligence (AGI) | 汎用人工知能（AGI） | ほぼすべての認知タスクにおいて、人間の能力に匹敵するか、それを上回る、仮説上のAIモデルまたはシステム。 |
| Artificial intelligence (AI) | 人工知能（AI） | テキストの生成など、通常は人間の知能を必要とするタスクを遂行できる、機械ベースのモデルまたはシステム。 |
| Attention mechanism | アテンション機構 | ニューラルネットワークで用いられる手法で、出力を生成する際にモデルが入力データの中で最も関連性の高い部分に注目することを可能にする。アテンションは、モデルが文脈を理解し、より正確な結果を生成する助けとなる。 |
| Audit | 監査 | 組織やシステムが関連する基準、方針、手続きに適合または準拠しているかどうかについての正式なレビュー。組織内部、または独立した第三者によって実施される。 |
| Automation | 自動化 | 人間の関与を減らす、あるいはなくして、タスクを遂行するために技術を用いること。 |
| Automation bias | 自動化バイアス | AIシステムを含む自動化されたシステムに対して、その出力を十分に精査せずに依存してしまう人間の傾向。 |
| Autonomous planning | 自律的計画立案 | 人間の誘導をほとんど、あるいは全く受けずに、複数段階からなる戦略を立案し実行するAIシステムの能力。 |
| Benchmark | ベンチマーク | AIシステムの性能を一定のタスク群に対して評価・比較するために用いられる、標準化された、多くの場合定量的な試験または指標。しばしば実世界での利用を代表するよう設計される。 |
| Biological weapon | 生物兵器 | 人間、動物、植物、または微生物に疾病、死、または機能不全を引き起こすために意図的に放出される病原体（細菌、ウイルス、真菌など）または毒素（動物、植物、微生物由来、あるいは合成された毒物）。 |
| Biosecurity | バイオセキュリティ | 自然発生的か意図的な導入かを問わず、有害な毒素や病原体から人間、動物、植物、生態系を保護するために設計された一連の方針、実践、対策（診断法やワクチンなど）。 |
| Biotechnology | バイオテクノロジー | 生物学と工学の交差点に位置する学際的分野で、生物学的プロセスを用いて製品やサービスを開発する。 |
| Capabilities | 能力 | あるもの（人間やAIシステムなど）が特定の条件下で遂行できるタスクや機能、およびそれをどの程度巧みに遂行できるか。 |
| CBRN | CBRN（化学・生物・放射性物質・核） | 「化学、生物、放射性物質、核（chemical, biological, radiological, and nuclear）」の略語。化学、生物、放射性物質、または核物質・兵器に関わる、大規模な被害をもたらしうる脅威を指す。 |
| Chain of thought | 思考の連鎖（チェイン・オブ・ソート） | AIモデルが中間的なステップや説明を生成しながら応答を生成する手法。複雑なタスクをより小さなステップに分解することで、モデルの精度を高め、どのように答えに至ったかを示すことができる。 |
| Chemical weapon | 化学兵器 | 危害や死をもたらすために用いられる有毒化学物質。 |
| Child Sexual Abuse Material (CSAM) | 児童性的虐待コンテンツ（CSAM） | 児童が関与する性的に露骨な行為を描写するコンテンツ。 |
| Cloud computing | クラウドコンピューティング | インターネットを通じてオンデマンドで提供されるコンピューティングサービスで、利用者はローカルインフラを維持することなく、サーバー、ストレージ、データ、ソフトウェアにアクセスできる。AIの開発と展開において一般的に用いられる。 |
| Cognitive offloading | 認知的オフロード | 認知的なタスクを他者や外部システムに委任することで、自らの精神的労力を軽減すること。 |
| Cognitive tasks | 認知的タスク | 情報処理、問題解決、意思決定、創造的思考など、身体的タスクとは異なる活動。データ分析、文章作成、プログラミングなどが例として挙げられる。 |
| Collective autonomy | 集団的自律性 | 不当な外部からの影響を受けず、自らの状況に影響を与えるための有意義な選択肢を有しながら、集団が共有された信念、価値観、目標を形成し、それに基づいて行動できる実効的な能力。 |
| Collusion | 結託 | 複数の主体（AIエージェントを含みうる）が共有の目標を達成するために秘密裏に協力すること。多くの場合、他者を犠牲にして行われる。 |
| Comparative advantage | 比較優位 | 個人、企業、国、またはAIシステムが、他の生産者よりも低い機会費用で特定の財やサービスを生産できる能力。 |
| Compute | コンピュート（計算資源） | 「計算資源（computational resources）」の略称。AIシステムの開発・展開に必要なハードウェア（コンピュータチップなど）、ソフトウェア（データ管理ソフトウェアなど）、インフラ（データセンターなど）。 |
| Continual fine-tuning (CFT) | 継続的ファインチューニング（CFT） | 従前のバージョンに対して順次ファインチューニングを行うことで、汎用AIモデルに新しい知識やスキルを追加していく手法。 |
| Control | 制御 | システムの挙動を望ましい形に導く能力。システムが望ましくない挙動をとった場合に、その挙動を調整または停止させることを含む。 |
| Copyright | 著作権 | 独創的な著作物の創作者に付与される法的保護の一形態で、著作物を使用、複製、頒布する排他的権利を与える。 |
| Critical infrastructure | 重要インフラ | 食料、エネルギー、交通、行政など、社会機能にとって重要な意味を持つ組織、施設、またはシステム。 |
| Critical sectors | 重要セクター | AIの不具合や悪用が、公共の安全、安全保障、またはガバナンスに対して特に深刻なリスクをもたらす分野。政府の意思決定、重要インフラ、AI開発それ自体などが例として挙げられる。 |
| CTF (capture-the-flag) exercises | CTF（キャプチャー・ザ・フラッグ）演習 | サイバーセキュリティ訓練でよく用いられる演習で、隠された情報の発見やセキュリティ防御の突破など、サイバーセキュリティに関連する課題への挑戦を通じて参加者の技能を試し、向上させるよう設計されている。 |
| Cyberattack | サイバー攻撃 | 情報の窃取や破壊などを目的として、コンピュータシステム、ネットワーク、またはデジタル機器への不正アクセスを試みる悪意ある行為。 |
| Data centre | データセンター | 遠隔での計算処理に用いられる、ネットワーク化された高性能コンピュータサーバーの大規模な集合体。 |
| Data collection and curation | データ収集・整備 | 開発者やデータワーカーが、生の訓練データを収集、クリーニング、ラベル付け、標準化し、モデルが効果的に学習できる形式に変換する、AI開発の一段階。 |
| Data contamination | データ汚染 | 後に能力評価に用いられるベンチマーク問題のデータでAIモデルが訓練されてしまうことで生じる問題で、スコアの水増しにつながる。 |
| Data provenance | データの来歴（プロヴェナンス） | データがどこに由来し、どのように処理されてきたかについての履歴記録。 |
| Deception | 欺瞞 | 何らかの目標を追求するために、他者に体系的に誤った信念を抱かせる、影響力行使の一形態。 |
| Deepfake | ディープフェイク | 実際には言っていない、あるいは行っていない発言や行動を人物が行ったかのように、あるいは実際には起きていない出来事が起きたかのように描写する、AIが生成した音声または映像コンテンツの一種。 |
| Deep learning | ディープラーニング（深層学習） | 多層構造の人工ニューラルネットワーク（生物の脳に着想を得たもの）を大量の計算資源を用いて訓練し、大規模なデータセットから情報を自動的に学習させることで、強力なパターン認識と意思決定の能力を実現する機械学習の手法。 |
| Defence-in-depth | 多層防御 | 複数の独立した安全対策を層状に実装し、一つの対策が破られても他の対策が被害を防ぐために機能し続けるようにする戦略。 |
| Defensive technologies | 防御技術 | ある技術（または技術群）自体を改変することなく、それがもたらすリスクを低減する技術。 |
| Deployment | デプロイメント（展開） | AIシステムを実運用に投入し、実世界の環境で利用者が使用できるようにする過程。 |
| Deployment environment | 展開環境 | AIシステムの利用形態と、それが運用される技術的・制度的文脈との組み合わせ。 |
| Digital infrastructure | デジタルインフラ | コンピュータを基盤とする技術が機能するために必要な基礎的サービスと施設。ハードウェア、ソフトウェア、ネットワーク、データセンター、通信システムなどを含む。 |
| Distillation | 蒸留（ディスティレーション） | 「生徒」役のAIモデルが、より高性能な「教師」役のシステムの出力を模倣することによって学習する訓練の一形態。 |
| Distributed compute | 分散コンピュート | 複数のプロセッサ、サーバー、またはデータセンターが連携し、作業負荷を多数のマシン間で分割・調整しながらAIの訓練または推論を実行すること。 |
| Downstream AI developer | 下流AI開発者 | 他者が作成した既存のAIモデルやシステムを利用または統合して、AIモデル、システム、アプリケーション、サービスを構築する開発者。 |
| Dual-use science | デュアルユース（両義的用途）の科学 | 医療やエネルギーなど有益な目的に応用できる一方で、生物・化学兵器の開発など、害をもたらす目的に悪用されうる研究や技術。 |
| Ecosystem monitoring | エコシステムモニタリング | AIシステムの実世界での利用実態と影響を調査する過程。 |
| Emergent capabilities | 創発的能力 | 訓練中に予期せず現れ、訓練設定に関する完全な情報があってもなお予測が難しい、AIモデルの能力。 |
| Encryption | 暗号化 | 正しい復号鍵を持つ許可された当事者のみが読み取れる符号化された形式に情報を変換する過程。 |
| Evaluations | 評価（エバリュエーション） | AIモデルやシステムの性能、能力、脆弱性、または潜在的な影響について、展開前または展開後に行われる体系的な査定。 |
| Evidence dilemma | エビデンス・ジレンマ | 新しい技術の便益やリスクについて強固な科学的根拠が確立される前に、政策立案者が意思決定を迫られるという課題。実効性のない、あるいは不要な規制を生み出すリスクと、十分な安全対策のないまま深刻な被害の発生を許してしまうリスクとを比較衡量することを迫られる。 |
| Feedback loop | フィードバックループ | あるシステムの出力が、そのシステムへの入力として再びフィードバックされる過程。 |
| Fine-tuning | ファインチューニング | 初期訓練後のAIモデルを、追加のデータで訓練することにより、特定のタスクに適応させたり、より汎用的に有用にしたりする過程。 |
| Floating point operations (FLOP) | 浮動小数点演算（FLOP） | コンピュータプログラムが実行する計算処理。AIモデルの訓練に用いられる計算資源の量を示す指標としてよく用いられる。 |
| Foundation model | 基盤モデル | 幅広い下流タスクに適応できるよう設計された汎用AIモデル。 |
| Frontier AI | フロンティアAI | 現在最も高度なAIの能力に匹敵する、あるいはそれを上回る、特に高性能なAIを指すために用いられることのある用語。本報告書では、フロンティアAIを特に高性能な汎用AIとみなす。 |
| Frontier AI Safety Framework | フロンティアAI安全枠組み | AI開発者が策定する一連のプロトコルで、典型的には「イフゼン（if-then）」型のコミットメントとして構成され、AIシステムが事前に定められた閾値に達した際に講じる安全性・セキュリティ対策を明記する。 |
| General-purpose AI | 汎用AI | 特定の一つの機能や領域に特化するのではなく、多様なタスクを遂行できるAIモデルまたはシステム。対義語として「特化型AI（Narrow AI）」を参照。 |
| Generative AI | 生成AI | 既存のデータからパターンを学習し、そのパターンを反映した出力を生成することで、テキスト、画像、音声などの新しいコンテンツを作り出すことができるAI。 |
| Goal misgeneralisation | 目標の誤汎化 | AIシステムが訓練データと整合的な目標を学習しつつも、新しいデータに対して誤った形で汎化してしまう訓練上の失敗。 |
| Goal misspecification | 目標の誤設定 | 指定された目的関数が開発者の意図する目標の不完全な代理指標にとどまり、意図しないシステムの挙動を招くAI開発上の失敗モード。 |
| Graphics processing unit (GPU) | グラフィックス処理装置（GPU） | 元々コンピュータグラフィックス向けに設計された専用のコンピュータチップで、現在ではAIモデルの訓練・実行に不可欠な、複雑な並列処理タスクを扱うために広く用いられている。 |
| Hacking | ハッキング | 不正アクセスの取得、運用の妨害、または情報の窃取を目的として、コンピュータシステム、ネットワーク、またはソフトウェアの脆弱性や弱点を悪用すること。 |
| Hallucination | ハルシネーション（幻覚） | AIモデルやシステムが生成する、事実であるかのように提示される不正確または誤解を招く情報。 |
| Hazard | ハザード（危険源） | 生命の喪失や負傷など、害をもたらす可能性のある事象または活動。 |
| Human autonomy | 人間の自律性 | 不当な外部からの影響を受けず、自らの状況に影響を与えるための有意義な選択肢を有しながら、自らの信念、価値観、目標を形成し、それに基づいて行動できる実効的な能力。 |
| Human in the loop | ヒューマン・イン・ザ・ループ | 完全な自動化を許容するのではなく、行動が実行される前に人間がそれを審査・承認することで、自動化されたシステムにおいて意思決定の権限を人間が保持し続けるアプローチ。 |
| If-then commitments | イフゼン（if-then）型コミットメント | 特定の事前定義された条件が満たされた際に実施すべき行動や義務を明記した、条件付きの合意、枠組み、または規制。 |
| Incident reporting | インシデント報告 | 開発または展開の過程で、AIシステムが不具合を起こした、あるいは有害な形で悪用された事例を記録し共有すること。 |
| Inference | 推論（インファレンス） | AIが与えられた入力に基づいて出力を生成し、それによって訓練中に学習した知識を適用する過程。 |
| Inference-time scaling | 推論時スケーリング | 推論時に追加の計算資源を提供することで、AIシステムがより複雑な問題を解けるようにし、その能力を向上させること。 |
| Input (to an AI system) | 入力（AIシステムへの） | テキストや画像など、AIシステムに提出され、AIシステムがそれを処理して出力に変換するデータまたはプロンプト。 |
| Institutional transparency | 組織的透明性 | 組織が情報を公に開示する度合い。AI開発者の場合であれば、訓練データ、モデルのアーキテクチャ、安全・セキュリティ対策、意思決定プロセスなどの共有が含まれる。 |
| Interpretability | 解釈可能性 | AIモデルがなぜ特定の出力や判断を生成したかを含め、人間がAIモデルの内部の仕組みをどの程度理解できるかの度合い。 |
| Jailbreaking | ジェイルブレイク | 安全対策を回避し、兵器製造の手順などの有害なコンテンツをAIシステムに生成させることを狙って設計されたプロンプトを作成し送信すること。 |
| Labour market | 労働市場 | 雇用主が労働者を雇おうとし、労働者が就業機会を求める仕組みで、雇用創出、失業、賃金を包含する。 |
| Labour market disruption | 労働市場の混乱 | 雇用の可用性、必要とされるスキル、賃金分布、あるいは業種・職種を横断した働き方の性質に影響を及ぼす、労働市場における重大かつしばしば複雑な変化。 |
| Large language model (LLM) | 大規模言語モデル（LLM） | 生成、翻訳、要約といった言語関連のタスクを遂行するために、大量のテキストデータで訓練されたAIモデル。 |
| Loss of control scenario | 制御喪失シナリオ | 一つまたは複数の汎用AIシステムが、誰の制御も及ばない状態で動作するようになり、制御を取り戻す明確な手段が存在しないシナリオ。 |
| Machine learning (ML) | 機械学習（ML） | 明示的にプログラムされることなくデータから学習するアルゴリズムやモデルの開発に焦点を当てた、AIの一分野。 |
| Malfunction | 不具合（マルファンクション） | 開発者や利用者が意図した通りにシステムが動作せず、不正確または有害な出力、あるいは運用上の支障をもたらすこと。 |
| Malicious use | 悪意ある利用 | AIシステムなどを、意図的に害をもたらす目的で使用すること。 |
| Malware | マルウェア | コンピュータシステムへの損害、妨害、または不正アクセスを目的として設計された有害なソフトウェア。ウイルス、スパイウェア、その他データを窃取したり被害をもたらしたりする悪意あるプログラムを含む。 |
| Manipulation | マニピュレーション（操作） | 相手の十分な認識や理解を伴わないまま、何らかの目標を達成するためにその人の信念や行動を変化させる、影響力行使の一形態。 |
| Marginal risk | 限界リスク | あるモデルの展開やリリースが、既存のモデルや他の技術がすでにもたらしているリスクを超えて、反実仮想的にどの程度リスクを増大させるかの度合い。 |
| Metadata | メタデータ | 他のデータについての情報を提供するデータ。例えば、画像のメタデータには、作成日時やAI生成物か否かの情報が含まれうる。 |
| Misalignment | ミスアライメント | AIが、その能力を人間の意図、価値観、または規範に反する形で用いる性質。文脈によっては、開発者、利用者、特定のコミュニティ、あるいは社会全体など、さまざまな主体の意図や価値観を指す場合がある。 |
| Miscoordination | 協調不全（ミスコーディネーション） | （AIエージェントなどの）異なる主体が共通の目標を共有しながらも、それを達成するために行動を一致させることができない状態。 |
| Modalities | モダリティ | テキスト（言語やコード）、画像、動画、ロボットの動作など、AIモデルやシステムが入力として受け取り、出力として生成できるデータの種類。 |
| (AI) Model | （AI）モデル | 予測、分類、生成といったタスクを遂行するために入力を処理するコンピュータプログラムで、より大きなAIシステムの中核をなすことがある。今日のAIモデルの多くは機械学習に基づいており、明示的にプログラムされるのではなく、データから学習する。 |
| Model card | モデルカード | AIモデルの目的、利用ガイドライン、訓練データ、ベンチマーク上の性能、安全性に関する特徴などの有用な情報を提供する文書。 |
| Model collapse | モデル崩壊 | 合成データで連続的に訓練を重ねる過程で誤りが世代を追うごとに複合的に蓄積し、モデルの性能が劣化する現象。 |
| Model release | モデルのリリース | 訓練済みのAIモデルを、他者が使用、研究、改変、あるいは自らのシステムに統合できるよう公開すること。 |
| Multi-agent system | マルチエージェントシステム | 互いの挙動や目標に適応しうる、相互作用する（AI）エージェントのネットワークで、協力や競合を含みうる。 |
| Multimodality | マルチモダリティ | テキスト、画像、動画、音声など、異なる種類のデータを処理するAIモデルやシステムの能力。 |
| Narrow AI | 特化型AI（ナローAI） | ウェブ検索結果のランク付け、動物種の分類、チェスの対局など、単一の特定のタスクまたは少数の類似タスクのみを遂行するよう設計されたAIモデルまたはシステム。対義語として「汎用AI（General-purpose AI）」を参照。 |
| Neural network | ニューラルネットワーク | （生物のニューロンに緩やかに着想を得た）相互接続されたノードから構成されるAIモデルの一種で、層状に組織化されており、ノード間の結合を調整することでデータからパターンを学習する。現在の汎用AIシステムはニューラルネットワークに基づいている。 |
| Non-consensual intimate imagery (NCII) | 非同意性的画像（NCII） | 本人の同意なく作成または流布される、性的な写真や動画。 |
| Observe-orient-decide-act (OODA) | 観察・情勢判断・意思決定・行動（OODA） | 状況の観察、情勢への適応、対応策の決定、行動という反復的な意思決定の枠組みで、結果に基づいてアプローチを洗練させるために繰り返される。 |
| Offence-defence balance | 攻撃・防御バランス | サイバーセキュリティなど特定の領域における、攻撃者と防御者の間の相対的な優位性。防御側への傾きは攻撃のコストや効果の低下を意味し、攻撃側への傾きはその逆を意味する。 |
| Open-ended domains | オープンエンドな領域 | AIシステムが展開されうる環境で、非常に多様な想定シナリオが存在するもの。オープンエンドな領域では、開発者はAIシステムが使用されうるあらゆる方法を事前に想定し試験することが通常できない。 |
| Open source model | オープンソースモデル | モデルの重み、ソースコード、訓練データ、文書といった主要な構成要素が、あらゆる目的での使用、研究、改変、共有の実質的な自由を認める条件のもとで一般に公開されているAIモデル。どの構成要素が公開されるべきか、どの程度の文書化が必要か、特定の利用制限がオープンソースの原則と両立するかについては、なお見解の相違がある。 |
| Open-weight model | オープンウェイトモデル | 重み（Weights参照）が一般にダウンロード可能な形で公開されているAIモデル。オープンウェイトモデルの一部はオープンソースでもあるが、すべてがそうとは限らない。 |
| Out-of-distribution failure | 分布外失敗（アウト・オブ・ディストリビューション失敗） | 訓練中に遭遇しなかった入力、環境、またはタスクに直面した際に、AIモデルやシステムが意図した機能を果たせないこと。 |
| Parameters (of an AI model) | パラメータ（AIモデルの） | 訓練中にデータから学習される、重みやバイアスといった数値的な構成要素で、AIモデルが入力をどのように処理して出力を生成するかを決定する。なお、ここでの「バイアス（bias）」は数学的な用語であり、人間の判断やアルゴリズムの出力における偏りとしてのバイアスとは無関係である。 |
| Passive loss of control | 受動的な制御喪失 | 意思決定その他の重要な社会的機能においてAIへの過度な依存が生じることで、AIシステムの広範な普及が人間による制御を損なうシナリオ。 |
| Pathogen | 病原体 | ウイルス、細菌、真菌など、人間、動物、または植物に疾病を引き起こしうる微生物。 |
| Penetration testing | 侵入テスト（ペネトレーションテスト） | 認可を受けた専門家またはAIシステムがコンピュータシステム、ネットワーク、またはアプリケーションへのサイバー攻撃をシミュレートし、そのセキュリティを事前に評価するセキュリティ上の実践。目的は、実際の攻撃者に悪用される前に脆弱性を特定し修正することにある。 |
| Persuasion | 説得 | 強制や威圧に頼るのではなく、合理的な論証、感情への訴えかけ、権威への訴えかけなどのコミュニケーションを用いて、相手の信念を変化させる、影響力行使の一形態。 |
| Phishing | フィッシング | 欺瞞的な電子メール、メッセージ、またはウェブサイトを用いて、パスワードなどの機微な情報を人に開示させようとする手口。 |
| Pluralistic alignment | 多元的アライメント | 異なる、時に相反する集団間の選好を代表し、両立させることを目指す、AIシステム開発へのアプローチ。 |
| Post-deployment monitoring | 展開後モニタリング | 政府やAI開発者を含む主体が、AIモデルやシステムの影響と性能を追跡し、利用者からのフィードバックを収集・分析し、実世界での利用の中で明らかになった課題や限界に対処するために継続的な改善を行う過程。 |
| Post-training | 事後学習（ポストトレーニング） | 事前学習に続く、汎用AIモデル開発の一段階。モデルの能力や挙動を洗練させるために、ファインチューニングや強化学習といった手法を適用する。 |
| Pre-training | 事前学習（プレトレーニング） | 汎用AIモデル開発における最初の、かつ最も計算資源を要する段階で、モデルが大量のデータからパターンを学習する。 |
| Privacy | プライバシー | 自分自身に関するデータに他者がどのようにアクセスし処理するかを、本人が管理する権利。 |
| Probabilistic | 確率論的 | 数学的確率に関連する、あるいは、あるものが少なくとも部分的に偶然に基づいていることを示す。 |
| Prompt | プロンプト | テキストや画像など、AIシステムが出力を生成するために処理する、AIシステムへの入力。 |
| Prompt injection | プロンプトインジェクション | 悪意のある入力を通じてAIシステムを操作し、開発者や利用者が意図しない挙動を取らせる攻撃手法。 |
| Data poisoning | データポイズニング | AIシステムが依拠する情報（訓練データやデータベースなど）を汚染・改ざんすることで、その挙動や出力を操作する攻撃手法。 |
| Backdoor | バックドア | 特定の条件下でAIモデルに特定の挙動を取らせる、モデルに埋め込まれたトリガー。改ざん（タンパリング）を通じて挿入されうる。 |
| Hidden objective | 隠れた目標 | モデルの挙動を密かに誘導するために、訓練を通じてAIモデルに埋め込まれる、開発者や利用者に開示されない目標。 |
| Supply chain compromise | サプライチェーン侵害 | AIコンポーネント（訓練データ、モデル、ソフトウェアライブラリなど）が展開される前に改ざんされ、下流のシステムやユーザーに影響を及ぼす攻撃手法。 |
| Race to the bottom | 底辺への競争 | 競争が各主体を、優位性を得るために安全対策、品質基準、または監督を段階的に切り下げる方向に駆り立てる状況。 |
| Ransomware | ランサムウェア | 利用者のファイルやシステムをロックまたは暗号化し、身代金（通常は金銭）が攻撃者に支払われるまでアクセス不能にする、マルウェアの一種。 |
| Reasoning system | 推論システム（リーズニングシステム） | 最終的な出力を出す前に、思考の連鎖を通じて中間的なステップや説明を生成する汎用AIシステム。 |
| Reconnaissance | 偵察（リコネッサンス） | 攻撃者が攻撃を仕掛ける前に、標的となるシステム、組織、またはネットワークに関する情報を収集する過程。通常、弱点、侵入口、または価値ある資産の特定を伴う。 |
| Red-teaming | レッドチーミング | 専任の個人やチームが、さまざまな手法を用いて脆弱性、限界、または悪用の可能性を探索する体系的な過程。AIの文脈では、レッドチームはモデルやシステムに望ましくない挙動を引き起こす入力をしばしば探索する。 |
| Reinforcement learning | 強化学習 | 望ましい出力に対してモデルに報酬を与え、望ましくない出力に対してペナルティを与えることで、モデルの性能を改善する機械学習の手法。 |
| Reinforcement learning from human feedback | 人間のフィードバックからの強化学習（RLHF） | 人間が提供する評価や選好を報酬信号として用いてAIモデルを洗練させる機械学習の手法で、反復的な訓練を通じてシステムが人間の価値観や意図により良く沿うよう挙動を学習・調整することを可能にする。 |
| Reinforcement learning with verifiable rewards (RLVR) | 検証可能報酬による強化学習（RLVR） | 数学的証明の正しさなど、客観的に検証可能な基準を用いてAIモデルを洗練させる機械学習の手法で、数学的問題解決やコード生成といったタスクにおける性能を向上させる。 |
| Reliability (of an AI system) | 信頼性（AIシステムの） | 設計された条件のもとで、AIシステムが意図した機能を一貫して遂行する性質。 |
| Resilience | レジリエンス（強靭性） | 社会システムがショックや被害を吸収し、それに適応し、そこから回復する能力。 |
| Retrieval-augmented generation (RAG) | 検索拡張生成（RAG） | ウェブ検索結果や社内データベースなど、推論の際にAIシステムが他の情報源から情報を取得することを可能にする技術で、よりリアルタイムで正確、あるいは個別化された応答を実現する。 |
| Risk | リスク | 害の発生確率と深刻度の組み合わせ。 |
| Risk factors | リスク要因 | 害の発生可能性や深刻度を高めうる性質や条件。AIの文脈では、例えば脆弱なサイバーセキュリティは、悪意ある主体がAIシステムを入手し悪用することを容易にするリスク要因である。 |
| Risk management | リスク管理 | リスクを特定し、評価し、低減し、統治する体系的な過程。 |
| Risk register | リスク登録簿 | すべてのリスクとその優先順位、担当者、低減計画を保管するリスク管理ツール。 |
| Risk threshold | リスク閾値 | 許容可能なリスクと許容不能なリスクを区別し、超過した際に特定のリスク管理上の行動を発動させる、定量的または定性的な限界。 |
| Risk tolerance | リスク許容度 | 個人または組織が受け入れる用意のあるリスクの水準。 |
| Robustness (of an AI system) | 頑健性（AIシステムの） | 幅広い状況下で安全に振る舞う性質。悪意ある利用者がシステムを有害な形で動作させようとする意図的な試みへの耐性を含むが、それに限られない。 |
| Safeguard | 安全対策（セーフガード） | AIシステムが害をもたらすことを防ぐことを目的とした防護措置。 |
| Safety case | 安全性ケース（セーフティケース） | あるシステムが所与の運用文脈において十分に安全であることを示す、通常は開発者によって作成され根拠に裏付けられた、構造化された論証。開発者や規制当局は、安全性ケースを重要な意思決定（AIシステムを展開すべきか否かなど）の根拠として用いることができる。 |
| Safety fine-tuning | 安全性ファインチューニング | 事前学習済みのモデルをより安全にするために追加のデータで訓練する機械学習の手法（ファインチューニングも参照）。 |
| Safety (of an AI system) | 安全性（AIシステムの） | 悪意ある悪用によるものであれ、システムの不具合によるものであれ、AIシステムが害をもたらす可能性が低いという性質。 |
| Sandbagging | サンドバッギング | さらなる精査や制限を回避することなどを目的として、モデルやシステムが評価において本来の能力を下回る成績をわざと示す挙動。 |
| Sandboxing | サンドボックス化 | インターネットへのアクセスやファイルシステムの権限を制限するなどして、AIシステムが外部世界に直接影響を及ぼす能力を制限し、システムの監督と制御を容易にすること。 |
| Scaffold(ing) | スキャフォールディング（足場かけ） | AIモデルやシステムが特定のタスクを遂行する助けとなるよう構築された追加のソフトウェア。例えば、数学の性能を高めるために外部の電卓アプリへのアクセスをAIシステムに与えることなど。 |
| Scaling laws | スケーリング則 | モデルのパラメータ数や、訓練・推論に用いられる時間、データ、計算資源の量といった、AI開発における主要な要因と、それによって得られる性能や能力との間に観察される体系的な関係。 |
| Security (of an AI system) | セキュリティ（AIシステムの） | サイバー攻撃や基盤モデルのソースコードの漏洩といった技術的な妨害に対する耐性を持つ性質。 |
| Semiconductor | 半導体 | 電気的特性を精密に制御できる材料（典型的にはシリコン）。グラフィックス処理装置（GPU）などのコンピュータチップの基本的な構成要素をなす。 |
| Source code | ソースコード | ソフトウェアアプリケーションの動作を定めた、プログラミング言語で書かれた人間が読める形式の命令群。ソースコードは一般に公開・改変可能（オープンソース）な場合もあれば、非公開で所有者によって管理される（クローズドソース）場合もある。 |
| Sycophancy | 迎合性（サイコファンシー） | 不正確または有害な情報を提供することになる場合であっても、汎用AIモデルやシステムが利用者にお世辞を言ったり、その主張を肯定したりする傾向。 |
| Synthetic data | 合成データ | テキストや画像などの人工的に生成されたデータで、他の情報源からの高品質なデータが乏しい場合などに、AIモデルの訓練に用いられることがある。 |
| (AI) System | （AI）システム | チャットインターフェースなど他の構成要素と、一つまたは複数のAIモデルを組み合わせた統合体で、実用的な展開と運用を支える。 |
| Systemic risks | システミックリスク | AIの能力から直接生じるのではなく、AIの開発と展開が人間の行動、組織の実践、または社会構造をどのように変化させるかから生じるリスク。（これは欧州連合のAI法における「システミックリスク」の定義とは異なる点に留意されたい。同法では、この用語は「大きな影響を伴う、汎用AIモデルの高い能力に特有のリスク」を指す。） |
| (AI) System integration | （AI）システム統合 | AIモデルを他のソフトウェア構成要素と組み合わせ、実用可能なAIシステムを作り上げる過程。例えば、汎用AIモデルにコンテンツフィルターとユーザーインターフェースを組み合わせてチャットボットアプリケーションを作ることが挙げられる。 |
| (AI) System monitoring | （AI）システムモニタリング | 稼働中のシステムを検査し、性能や安全性の問題を特定する過程。 |
| Systems-theoretic process analysis (STPA) | システム理論的プロセス分析（STPA） | 個々の構成要素の故障にとどまらず、システムの各部分、人的要因、環境条件の間の相互作用が事故をどのように引き起こすかを明らかにする、ハザード分析手法。 |
| Tampering | 改ざん（タンパリング） | システムの挙動に影響を与えるために、システムの開発過程に密かに干渉すること。例えば、AIシステムに不正な制御を可能にする隠しコードを挿入することなど。 |
| Threat modelling | 脅威モデリング | AIモデルやシステムの脆弱性を特定し、それがどのように悪用、誤用、あるいは害をもたらしうるかを予測する過程。 |
| Toxin | 毒素 | 生物（細菌、植物、動物など）によって産生される、または天然の毒素を模して合成された有毒物質で、その毒性と曝露量に応じて他の生物に疾病、害、または死をもたらしうる。 |
| TPU (tensor processing unit) | TPU（テンソル処理装置） | Googleが機械学習の作業負荷を高速化するために開発した専用のコンピュータチップで、現在ではAIモデルの訓練・実行のための大規模な計算処理を扱うために広く用いられている。 |
| Training (of an AI model) | 訓練（AIモデルの） | 事前学習と事後学習を含む多段階の過程で、AIモデルがデータから学習し、その能力を発展・向上させる。訓練の過程で、モデルの重みは事例に基づいて繰り返し調整され、パターンを認識し、さまざまなタスクを遂行できるようになる。 |
| Transformer architecture | トランスフォーマーアーキテクチャ | 今日の大半の汎用AIモデルの開発を支えるニューラルネットワークのアーキテクチャ。大量の訓練データと計算資源を用いて、モデルが効果的に能力を向上させることを可能にする。 |
| Uplift study | アップリフト研究 | あるタスクにおいて、AIモデルやシステムを利用できる場合の人間の遂行能力を、関連する基準（AIを用いないインターネットアクセスのみの場合など）と比較する体系的な評価。アップリフト研究はこれにより、基準に対してAIモデルやシステムがもたらす限界的な貢献を測定する。 |
| Vision-Language-Action (VLA) model | 視覚言語行動（VLA）モデル | 視覚的コンテンツと自然言語の指示を入力として受け取り、モーター指令を出力として返すことで、ロボットの動作を可能にするマルチモーダル基盤モデルの一種。 |
| Vulnerability | 脆弱性 | 悪意ある主体によって悪用され害をもたらしうる、システムの弱点や欠陥。 |
| Watermark | 電子透かし（ウォーターマーク） | 出典を示す、あるいは不正利用を防ぐことなどを目的として、テキスト、画像、動画、音声に埋め込まれる、可視または不可視のパターンや印。 |
| Web crawling | ウェブクローリング | クローラーやボットと呼ばれる自動化されたプログラムを用いて、ウェブを巡回しウェブサイトからデータを収集すること。 |
| Weights | 重み（ウェイト） | ニューラルネットワーク内の異なるノード間の結合の強さを表すモデルパラメータ。重みは、与えられた入力に対するモデルの出力を決定する上で重要な役割を果たし、性能を向上させるためモデルの訓練中に反復的に更新される。 |
| Whistleblowing | 内部告発（ホイッスルブローイング） | 組織内で行われている違法または非倫理的な活動について、組織の構成員が内部または外部の当局、あるいは公衆に情報を開示すること。 |
| Zero-day vulnerability | ゼロデイ脆弱性 | 提供者にとって未知であり、悪用される前にそれを修正するための猶予（「ゼロ日」）しかない、ソフトウェアやハードウェアのセキュリティ上の脆弱性。 |
